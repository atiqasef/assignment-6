Frequently Asked Questions

What is the difference between var, let, and const?
  • var: Function-scoped, can be redeclared and reassigned, hoisted but with undefined value.
  • let: Block-scoped, can be reassigned but not redeclared in the same scope.
  • const: Block-scoped, cannot be reassigned or redeclared. Must be initialized when declared.
What is the difference between map(), forEach(), and filter()?
  • map(): Returns a new array with transformed elements.
  • forEach(): Iterates over an array but does not return anything.
  • filter(): Returns a new array with elements that pass a condition.
What are arrow functions, and how are they different from regular functions?
  • Arrow functions (=>) provide a shorter syntax for writing functions.
  • They do not have their own this, so they inherit this from the surrounding scope.
  • They cannot be used as constructors (no new keyword).
How do JavaScript Promises work?
  • A Promise represents a value that may be available now, later, or never.
  • It has three states: Pending, Resolved (Fulfilled), or Rejected.
  • .then() is used for handling success, and .catch() for errors.
How do closures work in JavaScript?
A closure is when an inner function remembers variables from its outer function even after the outer function has finished executing.
